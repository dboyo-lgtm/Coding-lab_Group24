Kenyatta National Hospital project

Projexct Overview
This project is to build a secure, permission-governed environment and an analytical dashboard using Shell Scripting.

Group Roles & Contributions
- David Boyo Developed 'hospital_system.py' (core Engine)
- Arnold, Gary and David: created 'hospital_admin.sh' automated setup for the three sub tasks respectively
- David an Brian: created 'hospital_analysis' automated the set up for the two sub tasks respectively
- Samuella : created the 'hospital_archive.sh' which is for data archiving

hospital_system.py – The engine. Contains the core system logic that powers the hospital application, handling the main data operations and business rules.
hospital_admin.sh – Automated setup script covering milestones M1, M2, and M3, responsible for preparing the environment and bringing the required services online.
hospital_archive.sh – Archive script covering milestone M4, responsible for data archiving operations such as backing up and storing historical records.
hospital_analysis.sh – Analysis script covering milestones M5 and M6, responsible for generating reports and analytics from the system's data.
